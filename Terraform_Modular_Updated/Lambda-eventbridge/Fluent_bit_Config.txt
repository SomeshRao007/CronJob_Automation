QYUICK REFERANCE  GUIDE FOR FUTURE 


[SERVICE] Section
Flush: Interval in seconds to flush the data buffer (set to 300 second 5 minues).
Daemon: Determines if Fluent Bit runs as a daemon (Off means it runs in the foreground).
Log_Level: Sets the verbosity of logs generated by Fluent Bit (set to info).
Parsers_File: Specifies the path to the parsers configuration file (parsers.conf).

[INPUT] Section
Name: The type of input plugin used, here tail reads log files.
Path: Path to the log file being tailed (/var/log/cron.log).
Tag: A tag used to label incoming logs, useful for routing ( cron_jobs_DEV).
Refresh_Interval: How often (in seconds) Fluent Bit checks for new data in the log file (set to 100 seconds).
DB: Path to the database file used to track the log file's state, ensuring continuity after restarts (/var/log/flb_cron_jobs.db).
Mem_Buf_Limit: Maximum amount of memory that can be used for buffering logs (set to 15MB).
Skip_Long_Lines: Skips log lines longer than the buffer size (On means it skips them).

[OUTPUT] Section
Name: The type of output plugin used, here cloudwatch_logs to send logs to Amazon CloudWatch.
Match: Filters which logs to send based on their tag ( cron_jobs_DEV).
region: Specifies the AWS region where CloudWatch Logs will be accessed (us-east-1).
log_group_name: Name of the CloudWatch Log Group where logs are sent (1851_backend_DEV_CRON_logs).
log_stream_name: Specifies the log stream name (cron-job-logs).
auto_create_group: Automatically creates the CloudWatch Log Group if it doesnâ€™t exist (set to true).
log_retention_days: Number of days to retain logs in CloudWatch before they are automatically deleted (set to 30 days).
